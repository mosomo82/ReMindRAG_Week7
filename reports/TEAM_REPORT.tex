\section*{Division of Responsibilities}
Our team successfully executed Lab 7 by dividing the workload into three distinct, non-overlapping phases, allowing for parallel execution and individual accountability.

\subsection*{Tony Nguyen — Related Work Reproduction \& Core Pipeline Audit}
\textbf{Primary Ownership:} ReMindRAG Reproduction and Environmental Architecture
\begin{itemize}[leftmargin=*]
    \setlength{\itemsep}{1pt}
    \item \textbf{Repo Selection \& Setup:} Selected ReMindRAG as the related work, cloned the repository, and audited the initial environment.
    \item \textbf{Dependency Management:} Identified and fixed critical containerization blockers (e.g., missing CUDA package URLs, UTF-16 encoding errors in \texttt{requirements.txt}).
    \item \textbf{Reproducibility Auditing:} Authored the \texttt{REPRO\_AUDIT.md} document, logging 17 distinct reproducibility failures categorized by severity.
    \item \textbf{Automation:} Developed the \texttt{reproduce.sh} script (Linux/macOS) and \texttt{reproduce.ps1} (Windows/PowerShell) to allow single-command execution of the RAG pipeline.
    \item \textbf{Dockerization \& Cloud Deployment:} Created a multi-stage Dockerfile and debugged Streamlit Cloud deployment failures by resolving PyTorch PyPI wheel conflicts (\texttt{+cu126}).
    \item \textbf{Dependency Auditing:} Fixed a hidden tokenizer crash where NLTK failed to chunk Markdown tables, and mapped missing architectural dependencies (\texttt{einops} for Nomic MoE).
\end{itemize}

\subsection*{Daniel Evans — Evaluation Metrics \& Baseline Implementation}
\textbf{Primary Ownership:} Quantitative Benchmarking
\begin{itemize}[leftmargin=*]
    \setlength{\itemsep}{1pt}
    \item \textbf{Metric Expansion:} Designed the implementation plan to replace binary pass/fail RAG validation with robust quantitative metrics (ROUGE-L, BLEU, BERTScore).
    \item \textbf{Testing Infrastructure:} Developed the \texttt{smoke\_test.py} script to validate end-to-end traversal without requiring API keys.
    \item \textbf{Faithfulness Scoring:} Prototyped the prompt template required to verify that extracted knowledge graph entities are factually grounded in the source text.
\end{itemize}

\subsection*{Joel Vinas — Advanced Prompting \& Quality Enhancements}
\textbf{Primary Ownership:} LLM Reasoning and System Robustness
\begin{itemize}[leftmargin=*]
    \setlength{\itemsep}{1pt}
    \item \textbf{Reasoning Injection:} Designed the upgrade path for implementing Chain-of-Thought (CoT) and Self-Consistency methodologies into the entity extraction agent.
    \item \textbf{Cost Control:} Refactored the evaluation orchestrator (\texttt{eval\_LooGLE.py}) to allow dynamic switching of the judge model (e.g., overriding hardcoded \texttt{gpt-4o} with \texttt{gpt-4o-mini}).
    \item \textbf{System Improvements:} Integrated the \texttt{--seed} parameter globally to enforce strict deterministic randomness across all PyTorch and NumPy operations.
\end{itemize}
